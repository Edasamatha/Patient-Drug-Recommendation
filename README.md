# Patient-Drug-Recommendation

Patient Condition-Aware Drug Recommendation & Sentiment Analysis📌 OverviewThis project demonstrates a complete end-to-end data science and healthcare analytics workflow, from data preprocessing to model deployment. It integrates deterministic, condition-aware drug recommendation logic with NLP-driven sentiment analysis of clinical reviews to provide a holistic, patient-centered decision support tool.  📂 DatasetThe project utilizes two primary data sources:Structured Clinical Data: Includes patient attributes such as age, diagnosis, symptoms, allergies, and medical history used for drug matching.  Unstructured Clinical Reviews: Textual drug reviews sourced from healthcare forums to evaluate real-world drug effectiveness and public perception.  Dataset Tasks:Load and explore structured datasets using Pandas.  Standardize text data through an NLP pipeline.  Perform feature engineering and handling of missing values/outliers.  🛠️ Tools & TechnologiesPython: Primary language for data loading, cleaning, and model building.  Pandas & NumPy: Data manipulation and numerical computing.  Scikit-learn: Implementation of machine learning classifiers (Random Forest, Logistic Regression).  NLP (NLTK/spaCy): Text preprocessing, tokenization, and lemmatization.  Flask: Backend API development for handling recommendation and sentiment requests.  Streamlit: Interactive frontend dashboard for user input and visualization.  Hugging Face API: Inference for advanced sentiment analysis using transformer models.  🔄 Project StepsStep 1: Data Loading & EDA: Imported datasets and analyzed feature distributions and class imbalances using visualizations.  Step 2: Data Preprocessing: Standardized numerical features and encoded categorical variables; cleaned review text through tokenization and stopword removal.  Step 3: Machine Learning Modeling: Trained supervised models to predict suitable medications based on clinical features.  Step 4: Sentiment Analysis: Applied TF-IDF vectorization and classification models to categorize reviews as positive, negative, or neutral.  Step 5: Weighted Ranking Integration: Combined clinical prediction scores (70%) with sentiment scores (30%) to rank the final recommendations.  Step 6: Dashboard Development: Created a dual-panel Streamlit interface for structured patient input and live review analysis.  Step 7: Testing & Validation: Conducted unit, integration, and performance testing to ensure system reliability and accuracy.  📊 DashboardThe Streamlit interface features:Structured Patient Input: Fields for age, condition, allergies, and comorbidities.  Dynamic Recommendation: Top-ranked drug suggestions with associated confidence scores.  Sentiment Visualization: Summarized distribution of public perception for each medication.  Clinical Review Insights: Detailed breakdown of individual review sentiments and aggregated trends.  📈 Results & InsightsHigh Predictive Accuracy: Supervised models achieved balanced precision and recall for condition-based classification.  Contextual Ranking: Sentiment integration successfully lowered the rank of drugs with poor patient feedback despite high clinical suitability.  Actionable Decision Support: Provided a transparent framework for healthcare professionals to observe both clinical guidelines and patient experiences.  ▶️ How to Run the ProjectClone the Repository: git clone <repository-link>Install Required Libraries: pip install -r requirements.txt   Setup Environment: Rename .env.example to .env and add your HF_API_KEY.
